## 確定済みの技術方針(調査に基づく。変えるなら理由を明記)

1. **Three.js一択**(バンドル168KB gzip、実績)。Godot web exportは不採用(WebGL2互換のみ・モバイルクラッシュ報告)
2. **日本語テキストをWebGLに入れない**。troika-three-textは.woff2非対応+CJKのSDF生成時間に公開ベンチ無し=リスク。カード・UI文字はDOM層で従来通り
3. 性能予算: draw call <100(盤面のみなら余裕)、タイル/コマはInstancedMesh、モバイルはdevicePixelRatio 1〜1.5、ライトは実質0(エミッシブ+ベイク済みの見た目)
4. GLBパイプライン: Blender書き出し(+Y Up, Apply Modifiers)→ `gltf-transform optimize`(Draco+KTX2)。ただし盤面だけならプロシージャル生成(コード内でBoxGeometry)の方が軽い可能性大 — スパイクで比較
5. エージェントプレイ基盤(MCP合法手プロトコル)は描画非依存 → 3D化の影響なし
6. 撮影検証はMCPプレビューのスクショではなく**Playwright**(教訓L-005)。`tmp/shot-desk.mjs` にアートフェード待ち付きの雛形あり(untracked、消えていたら `tools/shot.mjs` を参考に再作成)

### ラウンド1(モックレンダ3枚への批評+コンセプト生成)

## 1. 方向性の合否

**5.5 / 10 —「SF卓」は成立。しかし「深宇宙サルベージ船のオペレーター卓」という固有性はまだ弱い。**

ティール／アンバー、暗い船内、浮遊盤面という大枠は正しいです。ただし現状は、サルベージ作業場というより「暗室に置かれた抽象的なホログラム卓」。円柱、カード、赤い多面体が何の装置なのか読めず、世界観よりプリミティブ形状が先に見えます。

3枚では `hero` が最も期待を感じさせますが、背景プレートとしては中央が騒がしすぎます。`main` はレイアウト確認用、`side` は空間構成の検証用で、完成画のカメラには向きません。

## 2. インパクト順の改善トップ5

### 1. 「サルベージ」を語る固有形状を入れる

最優先です。現在の黒い円柱とベージュの筒は用途不明で、安いプロトタイプ感を強めています。

- 左奥：伝票プリンター兼カード排出口を配置。幅 `0.9m`、高さ `0.35m`、投入口はカード幅の `1.2倍`
- 右奥：磁気封入キャニスター。直径 `0.28m`、高さ `0.42m`
- 右手前：手で引くクランプレバー。全長 `0.45m`、赤い握りを一点だけ使用
- 卓の外周：埋め込みレール、固定ボルト、焦げた切断痕、交換パネルを追加
- 背面窓または小型ポートから、回収対象の船骸シルエットを見せる

「印刷する・挿す・引く・封印する」という動詞が形から読める状態にしてください。

### 2. 黒潰れをやめ、暗部を3段階に分ける

現状は背景、家具、装置側面がほぼ同じ黒です。高コントラストではなく、情報が消失しています。

- ワールド背景：`#020609`
- 壁・大型家具：`#081217`
- 卓面：`#172123`
- 装置の暗面：`#10191B`
- 黒の最低値をリニア換算で `0.005` 前後ではなく、表示上 `RGB 6–12` 程度に確保
- Filmic/AgXなら `Look: Medium High Contrast` 程度。露出は現状から `+0.7〜1.0 EV`
- アンバー灯の照射範囲は卓面の左寄り `35〜45%` に限定し、背景全体を持ち上げない

暗くすることと、黒く潰すことは別です。

### 3. 卓面のCG的な鏡面感を抑え、素材差を作る

現状は卓面が大きな青灰色のプラスチック板に見えます。

- 卓面：`roughness 0.48前後 → 0.76〜0.82`
- 卓面：`metallic 0.5〜0.6相当 → 0.15〜0.22`
- Specular IOR Level：`0.25〜0.35`
- 法線ノイズ：実寸 `2〜8mm`、強度 `0.12〜0.2`
- 傷は全面均等にせず、手前の腕が触れる帯と装置周囲に集中
- 外周フレームだけ `metallic 0.65 / roughness 0.48` にして素材差を出す
- シアンの反射は面全体ではなく、縁に `5〜15mm` 幅で限定

粗さフィルタで均す方針は正しいですが、全素材を同じ粗さにするとミニチュアではなく未設定マテリアルになります。

### 4. カメラを「身体」にする

`main` は正面すぎて展示ケース、`side` は監視カメラに見えます。`hero` の距離感を基準にしつつ、DOM合成に適した安定した一人称へ寄せるべきです。

- レンズ：現在の望遠／正投影的な印象から `35〜40mm`
- カメラ高：卓面から `0.72〜0.82m`
- 下向き角：`12〜16°`
- 卓の手前縁を画面下から `8〜12%` に置く
- 左右に肘掛けまたは椅子の一部を画面幅の各 `3〜5%` だけ入れる
- 相手席は中央奥に置くが、輝度を卓面より `20〜30%` 低くする
- 水平線と卓の奥辺を完全平行にせず、ロール `0.5〜1°` 程度までのわずかな生々しさを許容

### 5. 色数と発光体を整理する

赤い多面体が現状の最強アクセントですが、意味も階層も不明です。背景プレート版では削除が正解です。

- ティール：`#2E8E91`
- 明るいシアンリム：`#73D8D5`
- アンバー：`#E7A85B`
- 警告色コーラル：`#D85C45`
- コーラルは画面面積の `0.5〜1.0%` 以下、一箇所に限定
- 発光面は左右合計 `3〜5個` まで
- シアンとアンバーの最大輝度比をおおむね `1 : 1.2`
- 星は現在の白点より輝度を `40〜60%` 下げ、数も半分以下
- 背面の巨大な黒い矩形は、単なる板ではなく厚さ `8〜12cm` の埋め込み機器としてベベル、ボルト、継ぎ目を加える

## 3. 背景プレート化する際の注意

中央のDOMが主役なので、背景は「周囲で世界を語り、中央では黙る」必要があります。

### 暗く保つ範囲

16:9画面を基準に、次の範囲を合成セーフゾーンにします。

- 横：`24〜76%`
- 縦：`35〜88%`
- 特に盤面直下の `横30〜70% / 縦42〜72%` は、輝度差と細部を最小化
- この領域の背景輝度は `15〜25%`
- UI直下の局所コントラストは、背景だけで `3:1` を超えない
- ボルト、傷、パネル境界などの高周波ディテールも中央から排除

単に暗くするだけでなく、中央を「低周波・低彩度・低コントラスト」にしてください。

### 視線誘導

背景側の情報は浅いU字に配置します。

- 左上：アンバーの物理ランプ
- 左端：プリンター／トグル群
- 右端：キャニスター／セル／レバー
- 奥中央：暗い相手席または背面装置
- 中央：空ける

左右の小道具から中央UIへ向かう斜線は作ってよいですが、卓面中央へ直接ライトを当てるとDOMと競合します。アンバー光は中央の左端で減衰させ、右側は細いティールリムだけにすると盤面が自然に浮きます。

### DOMとの合成

- DOMカードのドロップシャドウ方向を背景ランプと一致させる
- 背景側の被写界深度は弱くする。中央卓面はボケさせない
- プレートにブルームを焼き込みすぎない
- DOMの発光下にだけ、CSSで薄い色かぶりを足す
- 盤面非表示時にも卓が成立するよう、中央に焼き込みの疑似UIは置かない
- `main` のような真正面構図より、`hero` を少し引いた中間距離が適切
