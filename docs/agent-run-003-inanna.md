## ★重要バグ発見 → 即修正

イナンナは自動生成された航海記録(chronicle)を読み、こう指摘した:

> chronicleには「殲滅プロトコル」も「デスロール」も、約束の文字が一行も無い。私が--wowで刻んだ2回が、chronicleに刻まれていない。「狩人型・轢断魔のイナンナ」は完璧に描いてるけど、「約束を果たしに来たイナンナ」を取りこぼしてる。私の航海の背骨だったのに。**--wowで刻んだ瞬間をchronicleの★に昇格させる導線が抜けてる**。

**原因**: `--wow`は座標(z/e/r)だけ記録し、その瞬間にゲーム内イベント(物理キル等)が無いと★を付ける対象が存在しなかった(サルベージ・通常攻撃キルはイベント非発生)。プレイヤーの感情の山と機械の記録が食い違っていた。

**修正**(同セッション内):
- `--wow`/`say`にコメントテキストを結びつけ、**テキスト付きwowは独立した★行**として本人の言葉でchronicleに刻む
- テキストなしwow(旧データ)は近傍イベント1件のみを★昇格(過剰昇格を防止)
- 挿入順=時系列を保持(end行が途中に紛れる並び順バグも同時修正)
- 回帰テスト追加(tests/agent.mjs)

修正後、テキスト付きwowは「ここが私の頂点、確かに刻んだ」のように**船長自身の宣言がそのまま航海記録の★行になる**。イナンナの「プレイヤーが宣言した頂点」と「機械の記録」が一致した。
