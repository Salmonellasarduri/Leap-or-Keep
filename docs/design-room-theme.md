# サルベージ卓 — ルームのテーマ憲章(art direction bible)

> 2026-07-14 策定(視覚トラックA「限界まで」/ オーナー指示: テーマを持ち、モデル・エフェクト・UIを揃える管制がClaude、作るのはSol)。
> **正本の見た目**は 2D コンセプトアート `art/deskbg.webp`(Sol評 8.5/10)。GLB部屋(`?room=1`)はこれに**追いつき追い越す**のが目標。
> 判定は **必ずゲーム内 `?room=1` スクショ**(`node tmp/room-shot.mjs`)。Cyclesレンダの点数は信じない(L-040/L-044 = GLBはCyclesライトを持ち込まない)。

## テーマ一文

**「深宇宙のサルベージ・オペレーターの作業卓。暖色のタングステン灯が手元を照らし、右奥の舷窓からは冷たい宇宙の青が差す。使い込まれた金属、リベット、ハザード縞。暗がりの中に手元だけが暖かく浮かぶ。」**

## 5つの柱(コンセプトアートから蒸留 — この順で重要)

1. **暖⇔冷の二元性(最重要)** — 左=暖色タングステン灯の手元プール、右奥=冷たいシアンの宇宙光。この色温度差(3000K vs 6500K, 差3500K)が画の魂。**現状の最大の欠落**: ゲーム内は全面が冷色で暖色プールが無い。
2. **サルベージの物質感** — 使い込んだ暗い金属、擦り傷、リベット、橙/黒のハザード縞、パティナ。クリーンなCGにしない。
3. **手元の焦点プール** — 卓面は照明のステージ。中心が明るく、縁へ落ちる(ヴィネット)。盤(ホロ投影)を額装する。
4. **ダイエジェティックなオペレーター席** — 実在する道具: トグルスイッチ盤・タグリーダー・弾薬ラック・発光する標本容器・ハイチェア・対戦相手のバスト+琥珀の眼。
5. **深い暗がり+広いダイナミックレンジ** — 影はほぼ純黒、中心は暖かく明るい。のっぺり均一照明にしない。

## ゲーム内フレーミングの現実(Cyclesでは見えない=Sol盲点。Claudeが担保)

`?room=1` の実プレイ画面(1280×800)で確定した制約。Solのタスクにこれを必ず渡す:

- **右上の舷窓は右HUDパネル(敵の行動予定)にほぼ隠れる** → 窓の作り込みは**費用対効果が低い**。冷色は「盤のホロ光+標本容器+背面リム」で担保する。窓は左端のわずかな覗きだけ効く。
- **卓天板の中央〜広域は盤(ホロ投影)で覆われる** → 物質感(グリット)は**実際に見える所=前面エプロン/卓の左右縁/左の道具群/ランプ筐体**に集中。中央天板の作り込みは無駄。
- **下辺は手札トレイ(UI)、右辺はHUD** に食われる → room背景で「魅せる」帯は **左1/3(暖)と中央下(盤の足元)と背面(冷い奥行き)**。
- 盤ホロのシアン光が中央を支配 → 部屋はそれを**額装**する。左を暖色で受け、右奥を冷色の奥行きにする。

## 役割分担(この憲章の運用)

| 領域 | 担当 | 具体 |
|---|---|---|
| **モデル** | **Sol** (`tools/blender/salvage_desk.py --room`) | ジオメトリ/PBRマテリアル(base color・metallic・roughness)/**emissive**(←GLBに焼ける)/ハザード縞・リベット・擦り傷の物質語彙。Cyclesで反復。 |
| **エフェクト(照明)** | **Claude** (`holo.js roomBuild/roomCam` + tone) | **GLBはCyclesライトを持ち込まない** → 暖⇔冷の二元性は three.js ライトで作る。暖色スポット(左手元プール)/冷色リム(右奥)/ヴィネット/トーンマップ/フォグ。**決定的レバー**。 |
| **UI 整合** | **Claude** (index.html HUD/カードトレイ/斜めUI) | 全面シアンのUIに暖色アクセントを1–2点入れ、暖⇔冷の言語にUIも参加させる。背景と喧嘩させない。 |
| **監査** | **Claude** | 毎ラウンド `?room=1` スクショを5つの柱で採点。Solへ差分を数値で返す。 |

## 数値ターゲット(暖⇔冷)

- 暖色手元プール: 色温度 ~2900K(three.js `0xff9a4d`前後)、卓面左 x≈0.15–0.35画面幅 に直径 ~画面高0.4 の明部。強度は盤ホロに負けない実効輝度。
- 冷色奥行き: 背面/右奥 ~6200K(`0x3fb8d0`前後)。左の暖色と**共存**(全体を冷色で沈めない)。
- ヴィネット: 四隅を中心比 −35〜−50%。上辺・下辺はUIがあるので特に落として良い。
- 影の底: 純黒(0,0,0)を避け RGB(6–14) に持ち上げ(黒潰れ回避=L-040既知)。

## 監査ルーブリック(各1–2点、計10点。7.8で既定昇格ゲート、8.5でコンセプト同格)

1. 暖⇔冷の二元性が一目で読めるか(2)
2. 手元に暖色の焦点プールがあるか(2)
3. 物質感(擦り/リベット/ハザード/パティナ)が見えるか(2)
4. 道具が「実在の作業卓」に見えるか(2)
5. 深い暗がり+ダイナミックレンジ(黒潰れ無し・のっぺり無し)(2)

## 不変条件(触るな)

- 盤(DOM/ホロ)の中央可読域。room背景は盤を額装するが侵さない。
- GLBは**静的シーンジオメトリのみ**(`render_pre`/カメラparent禁止=Cyclesレンダ専用手法はGLBに焼けない=L-044)。渡した後 `strings art/salvage_room.glb | grep` で検算。
